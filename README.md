## 2. Техническая реализация

### 2.1. Модель данных
Для представления карты выбрана структура **списка смежности**. Это более эффективно по памяти для «разреженных» графов (городские дороги), где каждый перекресток связан лишь с несколькими соседними.

* **Узел (Node):** Объект, содержащий координаты $(x, y)$ на плоскости экрана.
* **Ребро (Edge):** Связь между двумя узлами.
* **Вес (Weight):** Динамический параметр, рассчитываемый по формуле Евклидова расстояния:
    $$d = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2}$$



### 2.2. Функционал приложения
Программа реализована на **Java (библиотека Swing)** и поддерживает три режима:

1.  **🖱️ Режим конструктора (ЛКМ):** Отрисовка дорожной сети поверх подложки Google Maps. Точки автоматически линкуются в сегменты.
2.  **🚩 Режим навигации (ПКМ):** Выбор `startNode` и `endNode` для расчета маршрута.
3.  **🔄 Режим редактирования (Колесо мыши):** Удаление вершин с автоматическим пересчетом индексов связей.

### 2.3. Алгоритм поиска
За поиск маршрута отвечает **алгоритм Дейкстры**.

* **Принцип:** Поддержка списка посещенных вершин и таблицы минимальных расстояний. На каждом шаге выбирается узел с наименьшим весом и проводится «релаксация» его соседей.
* **Сложность:** Благодаря `PriorityQueue`, сложность составляет $O((E + V) \log V)$, где $V$ — вершины, $E$ — ребра.

## 4. Контрольные вопросы

### 4.1. Что такое взвешенный граф?
Это структура, где каждому ребру сопоставлено числовое значение — **вес**.
* В навигации вес = расстояние, время или расход топлива.
* Если веса отсутствуют, поиск сводится к обычному подсчету количества шагов (ребер).

### 4.2. Как найти путь между вершинами?
1.  **BFS (Поиск в ширину):** Для невзвешенных графов (минимум пересадок).
2.  **Алгоритм Дейкстры:** Оптимален для взвешенных графов (без отрицательных весов).
3.  **Алгоритм A*:** Улучшенная версия с использованием эвристик для ускорения поиска.

### 4.3. Где применяются такие задачи?
* **Картография:** Google Maps, Яндекс.Пробки.
* **Сети:** Протоколы маршрутизации данных (BGP, OSPF).
* **Геймдев:** Поиск пути NPC (Pathfinding).
* **Финансы:** Поиск арбитражных сделок при обмене валют.
